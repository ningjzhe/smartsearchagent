## 运行方法
### 项目概览
本项目的目标是实现一个以agent为核心的多轮联网搜索增强和RAG问答的问答agent。

1.main.py是主程序，运行后可以提供一个运行交互式聊天界面。提问的规则显示在终端，用户可以手动启动联网搜索和RAG回答，若不启动，则由问答agent自主判断是否需要联网搜索回答和RAG回答。

2.search_agent.py是问答agent功能实现的核心模块，其中定义了AutonomousSearchAgent类。其实现的核心功能是处理用户的问题和多轮联网搜索和RAG增强回答问题能力。

3.web_searcher.py负责提供问答agent联网搜索的结果，通过将问答agent生成的查询输入Tavily的api后将结果返回给问答agent。

4.save_index.py负责建立RAG知识库，将目标目录的文件转化为向量库进行储存。

5.rag_agent.py定义了从向量库中寻找与问题相似的段落并且返回相应段落的RAG_agent,被search_agent.py调用。

6.results.txt展现了运行程序的两个例子，第一个例子展示了多轮搜索增强如何使得问答agent能够更加全面和深入地回答问题，第二个例子展示了RAG如何使得问答agent能够回答和私有知识有关的问题。

### 项目亮点

1.将RAG和联网搜索结合起来，使得问答agent既能够回答有关私有知识的问题，又能够回答需要实时信息的问题，扩大了问答agent能够回答问题的范围。

2.问答agent自主判断是否需要启用RAG或者联网搜索来回答问题，大大增强了问答agent的自主性和回答能力，不需要通过用户的硬（手动）要求来设置。在程序中，若用户没有手动设置联网搜索或者RAG，则问题将传给问答agent，由问答agent根据问题的性质来判断是否需要启动RAG或者联网搜索。

3.多轮搜索增强使得问答agent能够更加全面和深入地回答问题。本项目的一个特色在于实现了以问答agent为核心的多轮搜索增强。判断是否需要继续联网搜索，生成下一步搜索所需的查询，对得到的搜索结果作摘要，均由问答agent进行处理，因此是一个以问答agent为核心的多轮搜索增强。基本流程如下

A.每轮得到的搜索结果由大模型进行总结，得到搜索摘要，加入搜索摘要列表

B.将用户的问题和当前的搜索摘要列表传递给问答agent，由其判断是否需要进行下一步搜索。

C.当问答agent判断不需要进一步搜索时，或者达到最大搜索轮次时，问答agent根据最后的搜索摘要列表，针对用户的问题生成答案。

D.若问答agent判断需要进一步搜索，其将继续生成所需的查询，将查询传递给搜索引擎进行搜索，重复上述流程。



## 主要结果
1.程序的运行结果表明基本实现了项目目标。result.txt是程序运行结果的两个例子。问答agent能够通过多轮联网搜索增强来更加全面和深入地回答问题，也能够通过RAG来回答一些私有知识相关的问题

2.不足之处：

A.由于本次实验中通过向量化存储的知识库是一个我个人生活记录的知识库，因此一般而言RAG和联网搜索都是分别启动的,一般不能起到一个相互加强的效果，只是总的来说扩大了问答agent的能够回答的问题的范围。如果需要测试RAG和联网搜索共同增强问答agent回答问题的能力的话，需要向量化存储一个更有普遍性的知识库，例如物理学知识库。

B.尽管实现了有效的RAG增强，但是也暴露出当前RAG系统的一些缺陷。本次实验中，向量化存储的知识库是我个人生活记录的知识库，而当前的RAG系统通过查询与用户问题相似的向量然后重建文本并返回相应文本增强问答agent的回答能力。实验结果表明当前RAG系统不能很好的捕获与当前问题最直接相关的一些知识片段，可能是由于知识库本身具有的生活化的特性。下一步的改进应当要求优化Retrieve的实现，比如说要求问答agent针对提问生成一些可能的答案片段，再进行相应的检索，或许能优化retrieve的表现。
